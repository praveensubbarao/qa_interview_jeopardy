# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Repo Is

`qa_interview_jeopardy` has two distinct parts:

1. **Quiz app (repo root)** — an Express-backed, browser-based interview prep quiz with an AI-powered question generator. See `quiz.html`, `server.js`, `quiz-questions.js`, and `data/`.
2. **AI QE Design System (`AI QE Design System/`)** — the source-of-truth brand system for the AI QE platform: design tokens (CSS), self-hosted fonts, SVG brand assets, HTML specimen cards, a slide framework, and a React UI kit — all static files with no build pipeline.

The canonical brand spec is `AI QE Design System/uploads/Design Md Ai Qe Design System.pdf`.

## Quiz App (Root)

The root of the repo is an Express-backed quiz application separate from the design system.

| File / Dir | Purpose |
|---|---|
| `quiz.html` | Single-page frontend — quiz UI and admin "Add Questions" panel |
| `server.js` | Express server; serves `quiz.html` and the REST API |
| `quiz-questions.js` | Static fallback question bank (loaded client-side when server is offline) |
| `data/categories.json` | 17 category definitions with `id`, `label`, `icon`, `color`, and `keywords` |
| `data/questions/<category-id>.json` | Per-category question banks (683 total questions across 17 files) |

**Running:**

```bash
npm start       # kills any existing process on the port, then starts the server
npm run dev     # same with --watch (auto-restart on file changes)
```

**API endpoints:**

| Method | Path | Purpose |
|---|---|---|
| GET | `/api/categories` | Returns categories with live question counts |
| GET | `/api/questions` | Returns all questions from all category JSON files |
| POST | `/api/generate` | Generates + saves 1 question per targeted category via AI |
| POST | `/api/questions` | Manually adds a single question |

**AI providers** (set in `.env`, copy from `.env.example`):
- `OLLAMA_BASE_URL` + `OLLAMA_MODEL` → local Ollama (primary, no cost)
- `ANTHROPIC_API_KEY` → Claude (Anthropic SDK) — fallback when Ollama is unreachable

**Categories (17 total):** `ai-foundations`, `test-generation`, `self-healing`, `agentic-ai`, `data-defects`, `leadership`, `performance`, `technology`, `compliance-governance`, `jmeter`, `python-for-qa`, `javascript`, `playwright`, `aws-cloud`, `bdd-gherkin`, `api-testing`, `others`

**Question files** are in `data/questions/<category-id>.json`. Never edit `quiz-questions.js` directly — it is regenerated by `npm run build`.

## Design System — No Build Step

The `AI QE Design System/` folder has no bundler and no test runner. Open HTML files directly in a browser; serve with any static file server if cross-origin font loading is needed:

```bash
# Serve locally (from inside "AI QE Design System/")
python3 -m http.server 8080
# or
npx serve .
```

## Design Tokens (`colors_and_type.css`)

All tokens are CSS custom properties. Key semantic slots:

| Token prefix | Purpose |
|---|---|
| `--color-brand-*` | Intelligence Green (`#16945A`) family |
| `--color-accent-*` | Botanical Gold (`#D9B548`) family |
| `--color-slate-*` | Neutral scale |
| `--color-bg-*`, `--color-surface-*`, `--color-fg-*` | Light/dark theming |
| `--font-display` | Space Grotesk |
| `--font-body` | Inter |
| `--font-editorial` | Cormorant Garamond italic |
| `--font-mono` | IBM Plex Mono |

Dark theme is applied via `[data-theme="dark"]` on the root element.

## React UI Kit (`ui_kits/aiqe-app/`)

`Components.jsx` exports standalone components (no external dependencies beyond React): `Button`, `Badge`, `Sidebar`, `Topbar`, `RunList`, `RunDetail`, `AgentCard`, `MetricCard`. The companion `styles.css` references the design tokens; both must be loaded together. `index.html` is an interactive prototype — open it in a browser to see the full dashboard layout.

## Slide Framework (`slides/`)

`deck_stage.js` is a custom web component (`<deck-stage>`). Features: keyboard navigation, thumbnail strip, speaker notes, print-to-PDF. See `slides/index.html` for six template slide types.

## Specimen Cards (`preview/`)

24 HTML files — one per token/component category (e.g., `color-green.html`, `typography-body.html`, `badge.html`). Each is self-contained; open individually to inspect live token rendering.

## Brand Foundations (summary from README)

**Voice:** Calm, technically literate analyst — precise, not corporate; advanced, not jargon-heavy.

**Type scale (display → mono):** Space Grotesk → Inter → Cormorant Garamond italic → IBM Plex Mono.

**Iconography:** Lucide icon library, 2 px stroke, rounded geometry.

**Accessibility:** WCAG 2.1 AA, 4.5:1 minimum contrast, 44×44 px touch targets, green focus ring, `prefers-reduced-motion` respected everywhere.

**Layout:** 1200 px max-width container (marketing), 240–280 px sidebar (product), 4-column mobile grid.
